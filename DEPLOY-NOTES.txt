Kiosk System — deployment bundle (changed files only)
======================================================

1) UPLOAD / OVERWRITE
   - Extract the ZIP at the project ROOT (same level as `app/`, `config/`, `views/`, `bootstrap.php`).
   - Do not replace your `.env` or `config/database` credentials if already configured.

2) DATABASE (phpMyAdmin)
   - Select the correct database for the app.
   - Open the SQL tab and run the file:

     database/add_transactions_pending_and_payment.sql

   - NOTES:
     * If a COLUMN/INDEX already exists (e.g. you ran this before or the app created it), the whole ALTER may error.
       In that case, run only the missing ADD COLUMN / ADD INDEX statements one by one,
       or rely on the app’s auto-schema (CheckoutService) after deploy.
     * The app also applies some ALTERs at runtime (tolerant); production schema should still match when possible.

3) Files included in the ZIP (example list — may vary by release)
   - app/Controllers/DashboardController.php  (date range From–To; net sales; payment totals for range)
   - app/Controllers/Tenant/PosController.php  (card payment; FREE; pay pending)
   - app/Controllers/Tenant/ReportController.php
   - app/Core/Auth.php
   - app/Core/Router.php
   - app/Services/CheckoutService.php
   - bootstrap.php
   - config/route_paths.php
   - views/dashboard.php  (range filter; payment labels follow range)
   - views/layouts/app.php
   - views/tenant/pos/index.php  (cash amount field; quick amounts; FREE)
   - views/tenant/reports/index.php
   - views/tenant/transactions/index.php
   - database/add_transactions_pending_and_payment.sql

4) After deploy
   - Clear opcode cache if applicable (or restart PHP-FPM).
   - Hard refresh the browser (Ctrl+Shift+R) for new JS.

Thank you.
