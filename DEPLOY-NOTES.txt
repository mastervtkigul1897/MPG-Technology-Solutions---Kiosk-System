Kiosk System — deployment bundle (changed files only)
======================================================

1) I-UPLOAD / I-OVERWRITE
   - I-extract ang ZIP sa ROOT ng project (same level ng `app/`, `config/`, `views/`, `bootstrap.php`).
   - Huwag palitan ang `.env` o `config/database` credentials mo kung naka-setup na.

2) DATABASE (phpMyAdmin)
   - Piliin ang tamang database ng app.
   - Buksan ang tab na SQL at i-run ang file:

     database/add_transactions_pending_and_payment.sql

   - PAUNAWA:
     * Kung may COLUMN/INDEX na umiiral na (hal. na-run mo na dati o ginawa ng app), maaaring mag-error ang buong ALTER.
       Sa ganoong caso, i-run nang paisa-isa ang mga ADD COLUMN / ADD INDEX na kulang pa lang,
       o iasa na lang sa auto-schema ng app (CheckoutService) pagkatapos mag-deploy.
     * Ang app ay nag-a-apply din ng ilang ALTER sa runtime (tolerant); mas mabuti pa ring aligned ang schema sa production.

3) Mga kasamang binagong file sa ZIP
   - app/Controllers/DashboardController.php  (date range From–To; net sales; payment totals sa range)
   - app/Controllers/Tenant/PosController.php  (card payment; FREE once/day; pay pending)
   - app/Controllers/Tenant/ReportController.php
   - app/Core/Auth.php
   - app/Core/Router.php
   - app/Services/CheckoutService.php
   - bootstrap.php
   - config/route_paths.php
   - views/dashboard.php  (range filter; payment labels sumusunod sa range)
   - views/layouts/app.php
   - views/tenant/pos/index.php  (cash amount field; quick amounts; FREE)
   - views/tenant/reports/index.php  (tinanggal ang Daily/Weekly/Monthly/Yearly cards — nasa dashboard na)
   - views/tenant/transactions/index.php
   - database/add_transactions_pending_and_payment.sql

4) Pagkatapos mag-deploy
   - I-clear ang opcode cache kung meron (o i-restart PHP-FPM).
   - Hard refresh ang browser (Ctrl+Shift+R) para sa bagong JS.

Salamat.
